1. EXECUTIVE SUMMARY & SYSTEM GOAL
--------------------------------------------------------------------------------
Next-generation fantasy sports platforms demand sub-second latency, uncompromised security, and highly accurate AI predictions. TeamGenie AI satisfies this by using a multi-agent AI system (CrewAI paradigm) powered by Google Gemini and Anthropic Claude. The system scrapes data, feeds a 4-index RAG pipeline, and outputs risk-adjusted, budget-optimized 11-player teams in under 5 seconds.

3. THE FASTAPI BACKEND (apps/api) вЂ” DEEP DIVE & FILE RELATIONSHIPS
--------------------------------------------------------------------------------
The backend is built asynchronously to handle heavy LLM I/O and scraping.

A. The Entry Point: `apps/api/main.py`
- What it does: Bootstraps the FastAPI application. Sets up CORS, a global exception handler (Sentry integration), request ID injection, and initializes Redis caching on startup via lifespan events. 
- Connects to: `routers/` (to serve endpoints), `middleware/` (to wrap requests), and `services/cache_service.py` (for startup).

B. The Middleware Pipeline (Strict Chronology)
When a request hits `main.py`, it goes through these files in order:
1. `middleware/rate_limit.py`: Uses Redis to throttle per-IP requests to prevent abuse.
2. `security/ai_firewall.py`: Blocks obviously malicious signatures (SQLi, path traversal). The logic here intercepts malicious payloads before they parse.
3. `middleware/auth.py`: If the route isn't public, it cryptographically verifies the Supabase JWT.
4. `middleware/self_healing.py`: (Experimental mode) If an endpoint crashes, this catches the exception, captures stack traces, and prepares an AI-prompt to theoretically auto-suggest a code fix.
5. `middleware/error_handler.py`: Standardizes all unhandled exceptions into a clean JSON standard.

C. The Routers (apps/api/routers/)
- `team.py`: The heavy-lifter. Exposes `/generate` which triggers the multi-agent system.
- `match.py`: Contains WebSocket logic for live match data.
- `user.py`: Profile state and GDPR data export logic.
- `auth.py`: Login/Register, directly calling `auth_service.py`.

D. The Services (apps/api/services/)
These are stateless singletons containing business logic, kept out of routers:
- `cache_service.py`: Upstash Redis wrapper.
- `scraper_service.py`: Uses Playwright. If `cricbuzz.com` changes its CSS and crashes the scraper, `_self_heal()` triggers Claude to read the DOM error and suggest a new CSS selector.
- `rag_service.py`: The brain's memory. It uses `asyncio.gather()` to query 4 vector indexes (Player Stats, Match History, Venues, News) in parallel in <300ms, reranks results, and asks Gemini to synthesize the context.
- `ai_service.py`: The orchestrator of the multi-agent execution (detailed below).
- `auth_service.py`: A thin wrapper around the Supabase python SDK.

E. DB Connections (apps/api/db/connection.py)
- All connections (Turso, Supabase, Pinecone, Redis) are housed here.
- Why it matters: We employ the `tenacity` library here to provide automatic exponential backoff retries. Cloud databases disconnect; this file ensures the API doesn't crash on a transient network blip.

4. THE MULTI-AGENT SYSTEM (packages/ai & apps/api/services/ai_service.py)
--------------------------------------------------------------------------------
The request flow to build a team is orchestrated in `ai_service.py`, using configurations mapped out in `packages/ai/agents.py`.

How they talk to each other:
1. The user requests a team. `ai_service.py` fetches the raw player pool from Turso DB.
2. It parallelizes Agent 1 and Agent 2:
   - Agent 1 (Budget Optimizer - Gemini): Runs an integer-linear programming approach (simulated greedy logic currently) to maximize raw points under the в‚№100 limit. 
   - Agent 2 (Differential Expert - Gemini): Asks the RAG index to find 'hidden gems' (high upside, under 25% ownership).
3. Wait for 1 & 2 to finish, then trigger Agent 3:
   - Agent 3 (Risk Manager - Claude Haiku): Takes the output of 1 & 2, runs variance math, and decides who gets the Captain/Vice-Captain badges based on the user's risk profile parameter (safe/balanced/aggressive).
4. `ai_service.py` merges this into a finalized array of 11 `PlayerModel` (defined in `apps/api/models/team.py`) and returns it.

5. FRONTEND & MOBILE (apps/web & apps/mobile)
--------------------------------------------------------------------------------
- Shared logic: Both use `packages/shared/types.ts` so if the backend changes a schema, the frontend strictly catches the type error at compile time.
- Next.js 14 (`apps/web`): Server-side rendered for SEO. Heavy use of Framer Motion (`page.tsx`) to create a "wow" factor glassmorphism UI. We use `next.config.js` to enforce strict Security Headers to block XSS and clickjacking.
- React Native (`apps/mobile`): Uses Expo 52. Configured to explicitly share the UI state mentality of the web application through `NativeWind` (Tailwind for mobile).

--------------------------------------------------------------------------------
6. DATABASES & INFRASTRUCTURE
--------------------------------------------------------------------------------
- Primary Transactional DB: Turso (LibSQL). Selected because it mirrors SQLite at the edge, offering 0ms latency for edge-reads. `db/migrations/001_initial_schema.sql` defines 5 highly indexed tables.
- Vector DB: Pinecone (Player embeddings pushed using `sentence-transformers` via `packages/rag/embeddings.py`).
- Auth DB: Supabase (PostgreSQL under the hood).
- Edge Compute: `infra/cloudflare-worker.ts`. This script sits *in front* of the Vercel/Render servers. It checks `CF-IPCountry` and blocks requests from restricted Indian states explicitly before the request ever touches our servers.
- K8s Deployment (`infra/kubernetes/deployment.yaml`): Houses the Horizontal Pod Autoscaler (HPA), spinning up copies of the API image if CPU > 70%.

8. CONCLUSION
--------------------------------------------------------------------------------
This architecture is extremely defensive. At every boundaryвЂ”edge proxies, API middlewares, database connection retries, type-shared frontends, and self-healing Playwright scriptsвЂ”we have implemented fail-safes. The multi-agent topology correctly segregates concerns so that hallucination risk is isolated to the differential agent, while mathematical truths are strictly governed by the budget-optimizer integer logic.
